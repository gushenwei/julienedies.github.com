最近做的一个项目，后端只提供API接口，用于输出json数据，页面渲染完全由前端处理。

由于初期考虑不完善，后期遇到一些问题。其中之一是将html页面中的js/css引用添加时间戳或版本号。手动添加或修改肯定是不现实的。

后来想到了ant，果然圆满解决问题。

ant是基于java的任务构建工具（即通过可配置的程序组件自动完成日常开发中琐碎重复的任务，譬如单元测试，文件压缩合并打包，代码检查等）。

之前jQuery就是用其进行项目构建的，包括运行单元测试，代码检查，文件合并，代码压缩等。不过现在已经转用grunt，grunt是基于node平台，完全由javascript实现，而且专注于js开发的任务构建工具。

以下是此次特定任务的ant配置代码：

build.xml


<?xml version="1.0"?>

<project name="release" default="add-version-nu" basedir=".">
    
    <property file="build.properties"/>
    
    <!-- add version for js/css reference in html -->
    <target name="add-version-nu" depends="init">
        <replaceregexp byline="true" encoding="utf-8" flags="igm">
            <regexp pattern="${jsCssRefRegexp}"/>
            <substitution expression="\2\?t=${timestamp}"/>
            <fileset dir=".">
                <include name="**/*.html"/>
				<exclude name="**/*.js" />
            </fileset>
        </replaceregexp>
    </target>
    
    <target name="init">
        <tstamp>
            <format property="timestamp" pattern="yyyyMMddHHmmss"/>
        </tstamp>
    </target>
    
</project>

build.properties

lib.dir = lib
release.dir = release
htmlcompress.dir = compressed
temp.dir = temp

charset=UTF-8
js.dirName=js
js.dir=${root.dir}/${js.dirName}
js.concate.file=scripts_${build.version}.js
js.suffix=-min.js
css.dirName=css
css.dir=${root.dir}/${css.dirName}
css.concate.file=styles_${build.version}.css
css.suffix=-min.css
img.dirName=img
img.dir=${root.dir}/${img.dirName}
sprite.png.depth=AUTO
sprite.logLevel=INFO
sprite.suffix=-sprite
sprite.png.ie6
jstestdriver = ${lib.dir}/JsTestDriver-1.3.5.jar
jstestdriver.port = 9876
jstestdriver.browser = "C:\\Program Files (x86)\\Google\\Chrome\\Application\\chrome.exe",\
"C:\\Program Files (x86)\\Opera\\opera.exe",\
"C:\\Program Files (x86)\\Mozilla Firefox\\firefox.exe",\
"C:\\Program Files\\Internet Explorer\\iexplore.exe"
jstestdriver.config = conf.yml
jstestdriver.output = results

jsCssRefRegexp = ((assets/(css|js)/(base|helper|doc)\.(css|js))([?][t]=[0-9]+)?)
